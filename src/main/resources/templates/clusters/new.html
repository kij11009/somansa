Setup Guide
1. Create Service Account
kubectl create serviceaccount k8s-doctor-readonly -n default kubectl create clusterrolebinding k8s-doctor-readonly-binding \ --clusterrole=view \ --serviceaccount=default:k8s-doctor-readonly
2. Generate Token
kubectl create token k8s-doctor-readonly --duration=8760h

Duration 8760h = 1 year

3. Get API Server URL
kubectl config view --minify -o jsonpath='{.clusters[0].cluster.server}'
Security: K8s Doctor uses read-only access via the view ClusterRole.
